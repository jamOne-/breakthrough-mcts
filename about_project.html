About project


Game rules

Initially, the two front and the two back rows are filled with pieces. Choose a player to go first; then play alternates, with each player moving one piece per turn. A piece may move one space straight or diagonally forward if the target square is empty. A piece may move into a square containing an opponent's piece if and only if that square is one step diagonally forward. The opponent's piece is removed and the player's piece replaces it. Note that capturing is not compulsory, nor is it "chained" as in checkers. The first player to reach the opponent's home row — the one farthest from the player — is the winner. If all the pieces of a player are captured, that player loses. A draw is impossible because pieces can only move ahead (or be captured), and the piece closest to the opponent's home row always has at least one forward diagonal move available.
(source wikipedia.org)


Player types

  • Human — no computer player here; you just play the game locally using mouse.
  • Random — the weakest computer player, he just makes random move.
  • MinMax — better computer player, he minimizes the possible loss for a worst case scenario. His search depth can be set, which means how many moves forward he can analyze. For example, if set to 5, the player will analyze his 3 moves and opponent's 2 (all of his possible moves, then all posibilities of response of the opponent, then player's responses and so on).
    Note that the higher the number is, the more time it takes to calculate the move.
  • MCTS — computer player, which makes analysis of the most promising moves, expanding the search tree based on random sampling of the search space. His analysis is based on many playouts. Player's thinking time can be set — the higher this number is, the more playouts he can analyze.
    Thinking time is set with an error of 0-500ms.
  • MCTS asm.js — 3-4 times faster version of MCTS (uses asm.js optimalizations).


Algorithms used

MinMax

Instead of classic MinMax, I've implemented Negamax with alpha beta prunning.

Monte-Carlo Tree Search

Monte-Carlo Tree Search with UCT (Upper Confidence Bound 1 applied to trees) and with playout heuristic. They aren't totally random, with 99% chance, if there's a capturing move, this move is chosen. If there's more than one capturing move, with 99% chance, the one with enemy piece standing the furthest from its base is chosen.
